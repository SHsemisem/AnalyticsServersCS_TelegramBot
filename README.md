# Monitoring LOG

Telegram-бот для ежедневных и ручных отчетов по Counter-Strike 1.6 серверам.
Бот подключается к FTP сервера, скачивает логи AMX Mod X, разбирает их и отправляет результат в Telegram.

Поддерживаются два типа отчетов:

- `mapstats` - график среднего онлайна по картам за выбранную дату.
- `analyzer` - таблица источников мониторинга/подключений за выбранную дату.

## Что делает бот

- Загружает лог-файлы с FTP по шаблонам путей.
- Поддерживает несколько серверов в `servers.json`.
- Для каждого сервера может отправлять несколько отчетов.
- Строит PNG-график онлайна по `mapstats`.
- Формирует таблицу по `analyzer`.
- Отправляет отчеты автоматически каждый день в заданное время.
- Позволяет запросить отчет вручную через меню Telegram.
- Запоминает последний ручной выбор пользователя и предлагает повторить его одной кнопкой.

## Требования

- Python 3.12 или совместимая версия Python 3.
- Доступ к Telegram Bot API.
- FTP-доступ к серверу Counter-Strike.
- На игровом сервере должны создаваться логи `mapstats` и/или `analyzer`.

Зависимости указаны в `requirements.txt`:

- `python-telegram-bot`
- `matplotlib`
- `numpy`
- `pytz`
- `python-dotenv`

## Установка

Создайте виртуальное окружение и установите зависимости:

```bash
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Для Windows активация окружения обычно выглядит так:

```powershell
venv\Scripts\activate
pip install -r requirements.txt
```

## Настройка `.env`

Создайте файл `.env` в корне проекта. Минимальная конфигурация:

```env
BOT_TOKEN=123456:telegram_bot_token
CHAT_ID=123456789
TIMEZONE=Europe/Kyiv
REPORT_TIME=23:00
SERVERS_FILE=servers.json
```

Переменные:

| Переменная | Обязательная | Описание |
| --- | --- | --- |
| `BOT_TOKEN` | да | Токен Telegram-бота из BotFather. |
| `CHAT_ID` | да | ID чата, куда бот отправляет автоматические отчеты и ошибки. |
| `TIMEZONE` | нет | Часовой пояс для дат отчетов и расписания. По умолчанию `Europe/Kyiv`. |
| `REPORT_TIME` | нет | Время ежедневного отчета в формате `HH:MM`. По умолчанию `23:00`. |
| `SERVERS_FILE` | нет | Путь к JSON-файлу со списком серверов. По умолчанию `servers.json`. |
| `FTP_TIMEOUT` | нет | Общий FTP timeout в секундах, если не переопределен для сервера. По умолчанию `30`. |
| `FTP_HOST` | нет | FTP-хост для режима одного сервера без `servers.json`. |
| `FTP_USER` | нет | FTP-пользователь для режима одного сервера. |
| `FTP_PASS` | нет | FTP-пароль для режима одного сервера. |
| `FTP_PATH` | нет | Шаблон пути к логу для режима одного сервера. |
| `FTP_PASSIVE` | нет | Включение пассивного FTP-режима: `true`, `1`, `yes`. |
| `SERVER_NAME` | нет | Имя сервера для режима одного сервера. |
| `REPORT_TYPE` | нет | Тип отчета для режима одного сервера: `mapstats` или `analyzer`. |
| `REPORT_NAME` | нет | Название отчета для подписи в Telegram. |

Обычно удобнее использовать `servers.json`, потому что он поддерживает несколько серверов и несколько отчетов на сервер.

## Настройка `servers.json`

Пример конфигурации:

```json
[
  {
    "name": "Server1",
    "ftp_host": "127.0.0.1",
    "ftp_user": "ftp_login",
    "ftp_pass": "ftp_password",
    "ftp_timeout": 30,
    "ftp_passive": true,
    "reports": [
      {
        "type": "mapstats",
        "name": "Mapstats",
        "ftp_path": "/cs16/27015/cstrike/addons/amxmodx/logs/mapstats/mapstats_YYYY-MM-DD.log"
      },
      {
        "type": "analyzer",
        "name": "Analyzer",
        "ftp_path": "/cs16/27015/cstrike/addons/amxmodx/logs/analyzer/analyzer_YYYY.MM.DD.log"
      }
    ]
  }
]
```

Поля сервера:

| Поле | Обязательное | Описание |
| --- | --- | --- |
| `name` | да | Уникальное имя сервера. Используется в подписи отчета. |
| `ftp_host` | да | FTP-хост или IP-адрес. |
| `ftp_user` | да | FTP-логин. |
| `ftp_pass` | да | FTP-пароль. |
| `ftp_timeout` | нет | Timeout FTP-подключения для этого сервера. |
| `ftp_passive` | нет | Пассивный режим FTP для этого сервера. |
| `reports` | нет | Список отчетов. Рекомендуемый способ настройки. |
| `ftp_path` | условно | Старый режим одного отчета на сервер. Используется, если нет `reports`. |
| `report_type` | нет | Тип отчета в старом режиме: `mapstats` или `analyzer`. |
| `report_name` | нет | Название отчета в старом режиме. |

Поля отчета внутри `reports`:

| Поле | Обязательное | Описание |
| --- | --- | --- |
| `type` | да | Тип отчета: `mapstats` или `analyzer`. |
| `name` | нет | Название отчета в подписи Telegram. |
| `ftp_path` | да | Полный FTP-путь к файлу лога. |
| `ftp_timeout` | нет | Timeout только для этого отчета. |
| `ftp_passive` | нет | Пассивный режим только для этого отчета. |

В `ftp_path` поддерживаются шаблоны даты:

- `YYYY` - год, например `2025`.
- `MM` - месяц, например `10`.
- `DD` - день, например `27`.

Для `mapstats` обычно используется имя файла вида:

```text
mapstats_YYYY-MM-DD.log
```

Для `analyzer` обычно используется имя файла вида:

```text
analyzer_YYYY.MM.DD.log
```

## Форматы логов

### `mapstats`

Бот ожидает блоки такого вида:

```text
[2025-10-19 00:04:12] Карта: awp_india
  Уникальных игроков: 22
  Всего подключений: 30
  Отключений: 24
  Средний онлайн: 21 игроков
  Среднее время игры: 13 мин
----------------------------------------
```

Из этого отчета бот берет:

- дату и время записи;
- название карты;
- средний онлайн.

На графике дополнительно выводится сводка:

- средний онлайн за весь день;
- средний онлайн за день, вечер и ночь;
- карта с самым высоким средним онлайном днем;
- карта с самым сильным падением онлайна днем;
- пик онлайна;
- карта, которая встречалась чаще остальных.

### `analyzer`

Бот ожидает строки такого вида:

```text
L 10/27/2025 - 00:00:37: fullserver.org           STEAM_2:0:1111111111  IP   Player Nick
```

Из строки извлекаются:

- дата и время;
- источник мониторинга;
- Steam ID;
- IP;
- ник игрока.

В Telegram отправляется таблица с колонками:

- `site` - источник;
- `count` - всего событий;
- `unique` - уникальные события по паре Steam ID/IP.

Если таблица получается слишком длинной для сообщения Telegram, бот отправляет ее TXT-файлом.

## Запуск

После настройки `.env` и `servers.json` запустите бота:

```bash
source venv/bin/activate
python TelegrammBot.py
```

Бот работает через polling. При запуске он:

1. Загружает `.env`.
2. Загружает список серверов из `SERVERS_FILE`.
3. Регистрирует команды Telegram.
4. Запускает ежедневный планировщик.
5. Начинает принимать команды.

## Команды Telegram

| Команда | Описание |
| --- | --- |
| `/start` | Открыть меню ручного формирования отчета. |
| `/help` | Показать подсказку по работе с ботом. |
| `/cancel` | Прервать текущий диалог. |

Сценарий ручного отчета:

1. Отправьте `/start`.
2. Выберите дату: сегодня, вчера, позавчера или ручной ввод.
3. Выберите сервер или все серверы.
4. Выберите тип отчета: `Mapstats`, `Analyzer` или все типы.
5. Дождитесь сообщений с отчетами.

Дата при ручном вводе должна быть в формате:

```text
YYYY-MM-DD
```

Например:

```text
2025-10-27
```

## Автоматические отчеты

Автоматический отчет запускается каждый день в `REPORT_TIME` по часовому поясу `TIMEZONE`.

Например:

```env
TIMEZONE=Europe/Kyiv
REPORT_TIME=23:00
```

В этом случае бот каждый день в 23:00 по Киеву отправит отчеты за текущую дату.

Если при загрузке или обработке отчета возникает ошибка, бот отправляет сообщение об ошибке в `CHAT_ID`.

## AMX Mod X плагины

В каталоге `sma/` лежат исходники плагинов, которые создают логи для отчетов.

### `map_online_stats_logger.sma`

Пишет статистику карты в:

```text
addons/amxmodx/logs/mapstats/mapstats_YYYY-MM-DD.log
```

Также пишет отдельный лог по карте в:

```text
addons/amxmodx/logs/mapstats/<mapname>/YYYY-MM-DD.log
```

### `analyzer.sma`

Пишет analyzer-лог в:

```text
addons/amxmodx/logs/analyzer/analyzer_YYYY.MM.DD.log
```

Для работы использует `nvault`, `reapi` и данные мониторингов/хешей.

## Безопасность

- Не коммитьте `.env`.
- Не коммитьте реальные FTP-логи.
- Не публикуйте реальные FTP-пароли и токены Telegram.
- Если `servers.json` содержит реальные доступы, храните его приватно или используйте отдельный локальный файл через `SERVERS_FILE`.
- Для публичных примеров заменяйте IP, логины, пароли, Steam ID и ники на тестовые значения.

## Частые проблемы

### Бот падает при старте

Проверьте, что в `.env` заданы:

```env
BOT_TOKEN=...
CHAT_ID=...
```

Также проверьте, что `CHAT_ID` является числом.

### Бот не находит серверы

Проверьте `SERVERS_FILE`.
По умолчанию бот ищет:

```text
servers.json
```

Если этого файла нет, должны быть заданы `FTP_HOST`, `FTP_USER` и `FTP_PASS`.

### FTP зависает или долго отвечает

Увеличьте timeout:

```env
FTP_TIMEOUT=60
```

Или задайте timeout точечно в `servers.json`:

```json
"ftp_timeout": 60
```

Если FTP работает только в пассивном режиме, включите:

```json
"ftp_passive": true
```

### Отчет пустой

Проверьте:

- существует ли файл на FTP за выбранную дату;
- совпадает ли шаблон `ftp_path` с реальным путем;
- соответствует ли формат файла ожидаемому формату `mapstats` или `analyzer`;
- правильно ли выставлен `TIMEZONE`.

### График не строится

Для `mapstats` в логе должны быть блоки с названием карты и строкой:

```text
Средний онлайн: <число> игроков
```

Если таких строк нет, бот не сможет построить график.
