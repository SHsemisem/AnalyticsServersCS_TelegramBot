# Repository Guidelines

This repository hosts a Python Telegram bot that fetches Counter-Strike server logs from FTP, plots daily statistics, and pushes the report to a chat. Use this guide to align contributions with the existing workflow.

## Project Structure & Module Organization
- `TelegrammBot.py` holds the full runtime: log parsing, matplotlib plotting, FTP retrieval, and the scheduled Telegram job (`daily_job`).
- `requirements.txt` pins runtime dependencies; update versions deliberately and re-test before changes.
- `venv/` is a local virtual environment; do not commit its contents. Keep secrets in a local `.env` (BOT_TOKEN, CHAT_ID, FTP_* keys, TIMEZONE, REPORT_TIME, FTP_TIMEOUT).
- `servers.json` lists FTP credentials per server; бот шлёт отдельное сообщение для каждого элемента списка.

## Development Setup & Commands
```bash
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt         # install dependencies
BOT_TOKEN=... python TelegrammBot.py    # run the bot locally after configuring .env
```
For manual data checks, download an FTP log and run `python TelegrammBot.py` once; the job queue will post to `CHAT_ID` immediately on errors.
- Когда бот запущен в Telegram, отправьте `/test` в личку, чтобы мгновенно сформировать и получить отчёт.
- При добавлении серверов в `servers.json` указывайте полные FTP-пути (`mapstats_YYYY-MM-DD.log`) — бот отправит отдельный график на каждый сервер.

## Coding Style & Naming Conventions
- Follow PEP 8 with 4-space indentation and snake_case for functions such as `parse_log`.
- Mirror existing structure: keep parsing helpers pure, plotting in `make_plot`, and side effects inside async jobs.
- Prefer descriptive docstrings and inline comments only when behaviour is non-obvious (e.g., timezone conversions or FTP retries).

## Testing Guidelines
- Add automated tests under `tests/`, targeting `parse_log` and plotting helpers with synthetic log samples; organize fixtures under `tests/data/`.
- Use `pytest` and `pytest --maxfail=1` locally; strive for coverage on edge cases (missing averages, timezone handling, empty logs).
- When matplotlib output changes, compare generated bytes via image hashes rather than embedding binary fixtures.

## Commit & Pull Request Guidelines
- Write commits in imperative mood (`Add FTP retry backoff`) and keep scope focused—parsing, plotting, and scheduling changes should land separately.
- Pull requests must describe the problem, highlight risk areas (FTP access, plotting defaults), and note how to reproduce tests.
- Reference related issues or TODOs and include screenshots of generated plots when visuals are affected.

## Configuration & Security Tips
- Never commit `.env` or real FTP logs; provide redacted samples when necessary.
- Validate any new environment variables in `main()` with clear error messages to avoid silent misconfiguration.
- Keep `REPORT_TIME` as `HH:MM` in 24-hour format; the job scheduler uses it directly with the configured timezone.
- Maintain unique `name` values in `servers.json`; подпись отчёта использует это имя.
- Tune `FTP_TIMEOUT` (seconds) if отчёты «зависают» из-за долгого FTP-ответа, или задайте `ftp_timeout`/`ftp_passive` точечно в `servers.json`.
- Scrub personal data from screenshots or logs before sharing in issues or PRs.
